## Non-Obvious Local Dev Notes

### Prisma env loading

`prisma.config.ts` was updated so Prisma commands load `.env` and `.env.local`. This fixed the earlier issue where `prisma migrate deploy` only worked if `DATABASE_URL` was passed inline.

### Seed script

`prisma:seed` originally used the `tsx` CLI directly and hit an IPC/socket issue in this environment. It now uses:

```bash
node --env-file=.env --env-file-if-exists=.env.local --import tsx prisma/seed.ts
```

If `prisma:seed` fails, first confirm Postgres is running:

```bash
npm run db:up
docker compose ps postgres
```

### E2E auth bypass

The Playwright flow uses a test-only bypass via `E2E_AUTH_BYPASS=true` in `playwright.config.ts`. That bypass now short-circuits auth config before Entra env parsing and uses a narrow fallback user path in both:

- create action
- download route

This is specifically for local automation and should not be treated as a production auth pattern.

### Download route behavior

`/api/download/[requestId]` now:

- returns `401` if no user can be resolved
- returns `404` if the request is not owned by the current user or has no artifact
- serves the artifact with attachment headers if ownership is valid
- records `ARTIFACT_DOWNLOADED`

## Most Natural Next Steps

If development resumes later, the highest-value next areas are likely:

1. `My Apps` or request history
   So users can revisit previously generated packages without needing the immediate redirect page.

2. Better failure UX
   Friendly generation/download failure screens, support-reference display, and retry flows.

3. Download/page polish
   Show request metadata, status, generated time, and support reference on the success page.

4. Publishing hardening
   Replace the publish-profile-based Azure workflow with a stronger auth story, improve deployment verification, and decide how to package reproducible dependency installs for generated apps.

5. Template authoring improvements
   Better catalog ergonomics, more templates, clearer template validation rules, and stronger source-of-truth decisions between code and DB.

6. Production hardening
   Artifact retention, checksums on read, better audit surfacing, and a more intentional non-E2E local auth/dev strategy.
